## Part 1 - Unified canonical cleanup pipeline

### 1. Replace the current multi-button maintenance workflow
Replace the user-facing maintenance operations in Data with one primary action:
- **Clean Canonical Data**

Remove the need for the user to decide between:
- Top-Up
- Repair Interpolated Candles
- Reconcile Candles
- Enrich Candles

Those low-level concepts may remain behind the scenes or in advanced debug only if absolutely needed, but not in the main user flow.

### 2. Required cleanup pipeline behavior
The single **Clean Canonical Data** action must run this sequence in order:

#### Step 1 — Inspect existing canonical state
For the selected symbol, inspect:
- total rows
- real candle count
- interpolated candle count
- first date
- last date
- gap count if known
- health status

#### Step 2 — If no data exists, fetch everything available from the API
For symbols/timeframes that have not been run before:
- fetch all available historical data from the API
- store it into canonical candles
- do not rely on interpolation when real historical data is available

#### Step 3 — Detect missing ranges / gaps
Inspect existing canonical data and identify:
- missing candles
- broken ranges
- incomplete historical windows

#### Step 4 — Fetch missing real candles from the API
For every recoverable missing range:
- fetch real candles from API
- write them into canonical candles

#### Step 5 — Re-check existing interpolated candles
Inspect all existing interpolated candles in the selected symbol/range:
- for each interpolated candle/range, check whether real API candles now exist
- if real candles exist now, replace interpolated candles with real candles
- if real candles still do not exist, keep interpolated candles flagged

#### Step 6 — Only if still necessary, create interpolation as last resort
If there are still missing candles after all real API fetch attempts:
- create interpolated candles only where necessary to preserve canonical continuity
- mark them explicitly as interpolated
- do not pretend they are real

#### Step 7 — Re-run integrity
After real recovery and last-resort interpolation:
- recalculate total rows
- real candle count
- interpolated candle count
- recovered count
- unrecoverable interpolation count
- gap count if known
- first/last date

#### Step 8 — Run enrichment
After the canonical 1m candle dataset is repaired:
- run enrichment to derive all additional timeframes

#### Step 9 — Return final summary
Return a clear before/after result:
- rows before / after
- real rows before / after
- interpolated before / after
- fetched from API count
- replaced interpolated count
- unrecoverable interpolated count
- enrichment status
- export readiness status

### 3. Future cleanup behavior
Future cleanup runs must always:
- re-check gaps
- re-check interpolated candles
- replace interpolation with real candles if the API now has them
- then re-enrich

This must be explicit in both behavior and reporting.

### 4. Backend/API orchestration
If the backend does not already have a single route/handler for this full flow, add one.

The UI must not call broken or obsolete routes.
The broken Top-Up 404 path must be removed/fixed as part of replacing the whole workflow.

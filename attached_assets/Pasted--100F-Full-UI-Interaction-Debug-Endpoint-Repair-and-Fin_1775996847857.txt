You must identify the actual cause:
- missing backend route
- wrong route path
- wrong request method

- wrong response shape
- proxy issue
- wrong state refresh
- silent frontend error
- stale endpoint reference
- permission/mode gate
- or other real cause

### 2. No dead controls
Any visible control in the primary UI must:

### 3. No fallback logic
Do not add old-route fallbacks.
Do not silently call V2 endpoints if V3 endpoints fail.
If a route is wrong, fix it properly.

### 4. No hiding broken backend
If the backend route is missing or wrong, fix the backend route.

After fixing the known failures, inspect the whole primary UI for other broken or misleading controls and repair them in this same task.

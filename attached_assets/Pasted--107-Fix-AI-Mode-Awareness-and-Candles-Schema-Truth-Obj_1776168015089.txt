# BUG 1 — AI DISAGREE blocks valid paper trades

## What is happening
Live paper-mode signals with score around 62–65 are above the paper gate (>=60), but GPT often returns DISAGREE.

Root cause confirmed:
- `SignalContext` in `artifacts/api-server/src/infrastructure/openai.ts` has no `mode`
- the prompt shows all three gates:
  - paper >= 60
  - demo >= 65
  - real >= 70
- GPT is never explicitly told which mode it is evaluating
- so it can incorrectly apply demo/real logic to a paper signal

This makes valid paper signals get rejected unfairly.

## T002 — Make the prompt explicitly mode-aware

### File
`artifacts/api-server/src/infrastructure/openai.ts`

Update the prompt so GPT is told the exact trading mode and exact required gate for that mode only.

### Required prompt changes

In the CONTEXT section add:
- `Trading Mode: ${ctx.mode}`

In the SCORES section show:
- `Composite Score: ${ctx.compositeScore.toFixed(0)}/100`
- `Required gate for ${ctx.mode} mode: >=${ctx.mode === "paper" ? 60 : ctx.mode === "demo" ? 65 : 70}`
- `Gate result: ${ctx.compositeScore >= (ctx.mode === "paper" ? 60 : ctx.mode === "demo" ? 65 : 70) ? "PASS" : "FAIL"}`

### Critical instruction
In the STRICT EVALUATION CRITERIA block explicitly tell GPT:

- the mode gate has already been checked by the system
- GPT must NOT re-apply paper/demo/real score thresholds
- GPT should only judge:
  - setup quality
  - structural validity
  - whether the signal thesis makes sense

### Important
If the score already passes the current mode gate, GPT must not say DISAGREE only because it is below another mode’s gate.

#99 — GPT-5.1 + Context Retrieval Layer (FINAL)

What & Why

Upgrade the entire AI stack from gpt-4o to GPT-5.1 (with gpt-5-mini fallback), implement a context retrieval layer using text-embedding-3-large, and fix key UI issues in Move Calibration.

This ensures:
- AI Analysis works (no 403 errors)
- Move Calibration passes execute properly
- AI has real context from repo, strategies, DB schema, and calibration outputs

AI remains strictly research-only and must NOT enter the live trading loop.

- No gpt-4o usage anywhere in active code
- Central AI config defines:

  FALLBACK_MODEL = gpt-5-mini

- ai_context_embeddings table exists and can be populated
- Retrieval system returns relevant context for AI calls
- Calibration passes use retrieval context
- AI Analysis works without errors
- Move Calibration passes complete successfully
- Strategy selector is symbol-aware
- Window selector uses months (1–12 months)
- Behavior profile panel has a usable button (no raw API text)
- TypeScript builds clean

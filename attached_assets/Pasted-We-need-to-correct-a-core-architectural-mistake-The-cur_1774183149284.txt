We need to correct a core architectural mistake.

The current implementation wrongly ties signal generation to active trading modes.
That is not the intended design.

## REQUIRED ARCHITECTURE CHANGE

Separate the system into two independent layers:

### 1. Market Intelligence Layer (always-on when stream is on)
This layer must run even if no trading mode is active.

It must:
- consume live ticks
- compute features
- classify regimes
- run the 4 strategy families
- score setups
- create decision records
- populate the Signals / Decision Review page

This layer is for:
- research
- monitoring
- validation
- regime visibility
- signal collection

### 2. Execution Layer (mode-dependent)
This layer must only run if one or more execution modes are active:
- paper
- demo
- real

It must:
- consume the approved decisions from the market intelligence layer
- apply per-mode risk/capital rules
- place trades only for enabled modes

## REQUIRED BEHAVIOR

### If Data Stream is ON and all modes are OFF:
- system must still scan
- system must still produce regimes
- system must still generate decisions/signals
- Signals page must populate
- no trades should be placed

### If Data Stream is ON and one or more modes are ON:
- same signal engine runs once
- approved decisions are shared
- enabled modes may execute those same decisions using their own capital/risk settings

This means:
- the scanner must NOT depend on paper_mode_active / demo_mode_active / real_mode_active
- only trade execution depends on those flags

## FIX THE CURRENT BOOT LOGIC

Current root cause fix was wrong:
auto-enabling Paper mode just to get signals is not acceptable.

Required change:
- remove the dependency on active trading modes for scanner startup
- scanner should start when:
  - stream is running
  - scanner is enabled
- scanner should not require paper mode to be active

If no mode is active:
- scanner runs
- decisions are stored
- execution is skipped

## DECISION PIPELINE REQUIREMENT

The pipeline must be:

data stream -> features -> regime -> family routing -> scoring -> AI verification (if enabled) -> decision record

Then separately:

decision record -> execution layer -> paper/demo/real trade placement (only for armed modes)

## SIGNALS PAGE REQUIREMENT

The Signals / Decision Review page must show decisions generated by the market intelligence layer, even when no trading mode is active.

That page must NOT depend on trades existing.
